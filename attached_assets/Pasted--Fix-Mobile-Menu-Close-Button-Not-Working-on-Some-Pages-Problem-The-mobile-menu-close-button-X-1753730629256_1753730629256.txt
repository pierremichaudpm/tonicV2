# Fix Mobile Menu Close Button Not Working on Some Pages

## Problem
The mobile menu close button (X) works on `communiques.html` but NOT on other pages like `emplois.html`, even though they use identical HTML structure.

## Symptoms
- Mobile menu opens correctly on ALL pages when clicking hamburger button
- Close button (X) works ONLY on `communiques.html` 
- Close button does NOTHING on `emplois.html` and other secondary pages
- No JavaScript errors in console
- The `toggleMobileMenu()` function exists but isn't being called

## Current Implementation

The mobile menu HTML (identical on all pages):
```html
<button id="mobileMenuButton" onclick="toggleMobileMenu()" class="md:hidden">
    <!-- hamburger icon -->
</button>

<div id="mobileMenu" class="mobile-menu">
    <button onclick="toggleMobileMenu()" class="p-2 hover:bg-white/10 rounded-lg">
        <!-- X close icon -->
    </button>
</div>
```

## Task
Fix the close button so it works on ALL pages, not just communiques.html.

## Files to Check/Edit
1. `js/shared.js` - Contains navigation functions
2. `emplois.html` - Page where close button doesn't work
3. `communiques.html` - Page where close button DOES work (for comparison)

## Specific Actions Needed

1. **Diagnose why** `toggleMobileMenu()` works on communiques.html but not emplois.html
2. **Find the difference** between these pages that causes the issue
3. **Implement a fix** that ensures the close button works on ALL pages
4. **Test** that both open and close buttons work on:
   - emplois.html
   - communiques.html
   - Any other secondary pages

## Potential Issues to Check
- JavaScript load order differences
- Function scope issues
- Event handler conflicts
- Script errors that stop execution on some pages

## Success Criteria
- Mobile menu close button works on ALL pages
- No JavaScript errors in console
- Solution works without page-specific code

## Preferred Solution
Update `shared.js` with a robust implementation that:
1. Defines `toggleMobileMenu` globally
2. Uses event listeners instead of inline onclick
3. Handles all edge cases
4. Works regardless of script load order

Please implement and test the fix.
